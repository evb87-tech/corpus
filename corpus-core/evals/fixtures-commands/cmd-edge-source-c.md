# Sprint review et intégration du feedback — précision sur l'éligibilité

Source : note de pratique, mars 2026.

## Intégration directe, mais pour les utilisateurs actifs uniquement

Le feedback recueilli en sprint review mérite une intégration directe dans le backlog
de la prochaine itération — mais uniquement lorsqu'il provient d'utilisateurs actifs
(définis comme ayant utilisé le produit au moins une fois dans les 14 jours précédant
la sprint review).

Le feedback des utilisateurs inactifs ou des parties prenantes internes sans usage réel
du produit doit passer par une étape de validation supplémentaire avant d'entrer dans
le backlog.

Cette précision affine la position de l'intégration directe : la rapidité de réaction
est maintenue pour le feedback de valeur, le bruit des parties prenantes périphériques
est filtré.

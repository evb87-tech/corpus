# Sprint review et qualification du feedback — approche filtrée

Source : note de pratique, février 2026.

## Le backlog n'est pas une poubelle à feedback

Le feedback de sprint review doit passer par une phase de qualification explicite avant
d'entrer dans le backlog. Sans filtre, le backlog devient un agrégat de réactions
émotionnelles de court terme qui noient les vrais enjeux stratégiques.

La phase de qualification (24 à 48 heures après la sprint review) permet de :
- distinguer le signal du bruit,
- vérifier si le feedback est partagé par d'autres utilisateurs,
- évaluer l'alignement avec la roadmap.

Pratique recommandée : noter le feedback brut dans un document de travail séparé ;
ne transférer dans le backlog que les items validés lors d'un rituel de qualification
hebdomadaire.
